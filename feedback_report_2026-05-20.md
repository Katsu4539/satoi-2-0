## 1. 入口設計の根本的な見直し【最優先】

橋谷さんが最も強く繰り返した指摘。現状の入口は専門家視点で組み立てられていて、ユーザー目線になっていない。

- **「自分の状況」を最初に必ず入力させる選択肢を入口に置く**
  - がん種(肺/胃/乳/大腸…)+ ステージを最初に入力
  - ステージは 1未満 / 1.0 / 1.5 / 2.0 / 2.5 / 3 / 4 と細かく入力可
- **入力後は即座に「5年生存率/死亡率/全国患者数」をバンと提示**
  - 「同じ境遇の方がこれだけいます」「生存率はこれ」を一気に見せて自分ごと化
  - 「自分は最悪死んでもしょうがない」と覚悟する気持ちにも対応
- **ディシジョンツリーを"ゼロ"に近づける**
  - 入力情報で次の分岐が自動絞込されるよう設計
  - 入口でセグメントできていればシステムコストも下がる
- **最初の質問は最大2〜3個まで**
  - 質問が多いほど客は離れる

## 2. 患者UXの根本転換:「ユーザー目線」への翻訳

橋谷さん曰く「知らない人の意見が一番重要」。Katsujiさんが知っているがゆえに専門家目線で組み立ててしまっている、というのが核の指摘。

- **最初の問いは「僕は生き残れるんですか?」**
  - これに答える数字(生存率)を最初に見せる
- **生存率が低い場合の自然な動線は「家族のために何ができるか」**
  - 自分のための情報 → 家族のための情報 へ視線が移る
  - 鈴木のお父さん事例:肺がんで15年は稀少ケース、本人は「なんなんだこれは」の気持ち
- **「あなたの記録は全部残ってますよ」を明示的に伝える**
  - 親父さんエピソード:「お前に任せる」と言いつつ白米を食べる父 → 記録継続を見せて安心感を与える
  - 「前回の続きから」AI会話開始は良い実装、もっとフロントに出す
- **専門用語の地雷を踏まない**
  - 「ニボルマブ」「化学療法」「薬物療法」は素人にはわからない
  - クリック→展開で説明する仕組みを整える
- **感情の選択肢を入れる**
  - 「治療」一辺倒でなく「共存する」「家族と過ごす」など選択肢を並列に
- **治療法は「王道」を先に提示**
  - ステージ4・肺がんなら王道はすでに決まっている
  - 「もっと他の治療法を知りたい」で詳細展開
  - 自分で薬を選ばせない(=ベビーカー理論)

## 3. 「選択肢を絞る」絶対原則(ベビーカー理論)

橋谷さんが2-3回繰り返し言及している重要原則。

- **4種類 > 32種類**(海外で実証されたベビーカー販売実験)
- **素人には選択肢が多いほど判断不能 → 離脱**
- プロは選べるけど一般患者・家族は選べない
- 「楽に解決したい」がユーザーの本音
- **やめることから始める**(マツダ・ユニクロ流デザイン哲学)
- **Apple流「8個のデータ」モデル**
  - Appleが楽天と組んだ時:データは8個だけでいい、それで自信ある
  - 日本の会社は「多い方がいい」とやってシステムが肥大化する
- **現状のSATOIは「ナイスハブの塊」状態**
  - Phase1で大幅に削ぎ落とす
  - Phase2でチュートリアル形式に機能追加
  - チュートリアル設計はAIに任せられる

## 4. 家族・SDM・終末期視点の強化

- **家族のために何ができるかの動線を明確に**
  - 生存率を見た後、自然に家族視点に流れる導線
- **ACP(アドバンスケアプランニング)/ SDM の伴走機能は維持**
  - これは厚労省推奨内容、ちゃんと押さえている部分
- **家族登録は任意・別ID/PW**
  - パスワード使い回しNG(個人情報保護)
  - 家族IDを取得→家族メアドに飛ばす→家族が自分でPW設定
- **本人が言いたくないことを家族に伝える機能は不可**
  - 個人情報開示にあたる
  - SNSはオープンだから可、これは「家族だけ」だから個人情報開示
  - **動的同意(ダイナミックコンセント)**で対応するなら可能性あり
- **家族の口コミ効果は強力**
  - がんになった本人が家族に「この保険違うよ」と言うと自分ごと化する
  - ただし保険会社が家族にダイレクトアプローチはできない

## 5. 保険会社向けページの再設計【方針転換】

橋谷さんの最大の指摘ポイントの一つ。「ここまでやるの?」「あまり感じない」とかなり厳しめ。

### 5-1. 現状の課題
- 機能盛り込みすぎ(ナイスハブの塊)
- データ販売モデルだとマネーフォワードとライバルになる
- ターゲット配信・レポート・市場規模分析は保険会社の「興味薄い」領域
- セールスにはつながらない(がんになった人に保険売れない)

### 5-2. 収益モデルの方向
- **保険業界のM3(エムスリー)を目指す** ← 橋谷さんが繰り返し提示する原型
- 大手4社(日本生命・明治安田・住友・第一)なら月300万円は許容範囲
- サービス価値次第で月500万・1000万も可能
- AI活用で人件費換算でメリットを訴求(MR比較で30%等の単価設計)
- マネーフォワードとは差別化(がん特化で世界一のデータを目指す)

### 5-3. 機能の削ぎ落とし方針
- 一旦すべて削ぎ落として、本当に必要なものだけ残す
- 議論パートナー(第三者)を入れてディスカッション後に判断
- まず半分以下に減らす方針
