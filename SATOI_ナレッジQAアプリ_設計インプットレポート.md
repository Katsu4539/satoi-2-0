## 1. 背景・課題

SATOIプラットフォームには多数のナレッジレポートが蓄積されている。VC・投資家・保険会社・企業担当者がこれらを閲覧する際、「この数字の根拠は何か」「この戦略はどういう意味か」などの質問が頻発し、KatsuさんおよびAndoさんへの個別対応が都度必要になっている。

この対応コストを下げ、かつ「誰がどこに疑問を持っているか」というデータを取得・活用するために、ナレッジQ&Aアプリの構築を検討することとなった。

### 解決したいこと

- 資料の根拠・背景についての問い合わせをAIが自動回答できるようにする
- AIで答えられない場合のエスカレーション（Katsu・安藤さんへ通知）を自動化する
- ユーザーがどこに疑問を持っているかのデータを蓄積し、改善に活用する
- ブラウザ依存をなくし、アプリ（PWA）として運用できるようにする
- 既存のSATOIプラットフォームも同様にPWAアプリ化する

## 2. アプリ全体設計

### 2-1. ロール別ログイン（入口設計）

アプリ起動時、ユーザーは自分の属性に合ったロールを選択してログインする。ロールによって閲覧できる情報・質問できる深さ・表示されるコンテンツが変わる。

| ロール | 主なユーザー | 想定コンテンツ |
|---|---|---|
| 投資家・VC | VC・投資検討者 | 事業計画・バリュエーション・市場規模・Exitシナリオ |
| 保険会社 | 保険会社担当者 | がん保険給付フロー・診断書確認・申請ガイド |
| 企業リサーチ | 企業担当者・パートナー | サービス内容・臨床根拠・導入メリット |
| 患者・一般 | 患者・ご家族 | サービス概要・使い方・疑問点の問い合わせ |

### 2-2. 保険会社ロール（特記事項）

保険会社ロールは既存モック（Ver.2）に設計済みの「保険者窓口」機能を引き継ぐ。以下のフローで自動処理する。

1. ログイン時に登録保険情報を参照
2. 診断確定・病名を入力（確定診断書をアップロードして自動判定も可）
3. 給付対象・申請可能な保険を自動リストアップ（抜け漏れ防止）
4. そのまま申請フローへ移行できる導線

## 3. コア機能

### 3-1. ナレッジQ&A（AIチャット）

Claude Code管理下のローカルフォルダ（`./knowledge/`）に格納したナレッジレポート（.md / .txt）を読み込み、質問に対してAIが根拠付きで回答する。

| 項目 | 内容 |
|---|---|
| 回答品質 | ナレッジ本文のみを根拠に回答。出典ファイル名を必ず明示 |
| マルチターン | 会話履歴を保持。「先ほどの件について…」が使える |
| ナレッジ更新 | `node load_knowledge.js` を再実行するだけ |
| 未回答処理 | ナレッジに根拠がない場合は「資料に記載なし」と返答してエスカレーション促進 |

### 3-2. コンテキストQ&A（画面上の「？」ボタン）

プラットフォームの各ページに「？」ボタンを常時表示。タップした時点でどのセクションを閲覧中かを自動認識し、「このページについて質問する」という文脈付きで質問できる。

> ※ これにより「どのページで何を疑問に思ったか」のデータが自動的に取得できる。ヒートマップ的な分析への活用が可能。

### 3-3. エスカレーション機能

AIで解決できない場合のフロー。

1. ユーザーが「それでも分からない」→ 問い合わせボタンをタップ
2. 質問内容・閲覧ページ・ユーザー情報が自動入力された確認画面を表示
3. ユーザーが内容を確認・送信
4. KatsuさんとAndoさんのLINE（またはメール）に通知が届く
5. 問い合わせ内容・ユーザー情報・ページ履歴をDBに自動記録

### 3-4. データ取得・ログ管理

| 項目 | 内容 |
|---|---|
| 取得データ | 誰が / どのページで / 何を質問したか / AIが答えられたか |
| 保存先 | Supabase（PostgreSQL） |
| 閲覧方法 | 管理画面（Phase 3で実装）またはSupabaseダッシュボードで確認 |
| ヒートマップ | 各ページ・セクションへの質問数を集計→改善箇所の特定に活用 |

### 3-5. 同意取得（動的同意）

初回アプリ起動時に同意画面を表示。データ取得・活用についての同意が得られたユーザーのみが先に進める。患者さん向け配布の際も同様。

> ※ 動的同意の設計はSATOIのデータ情報銀行構想・EMPSe軸のエビデンス構築とも連動する重要要素。同意フローの詳細はCoworkで別途壁打ち予定。

## 4. PWAアプリ化

「毎回ブラウザ接続が必要」という課題をPWA（Progressive Web App）で解決する。ネイティブアプリのインストール不要でホーム画面への追加ができる。

| 項目 | 内容 |
|---|---|
| インストール方法 | ブラウザから「ホーム画面に追加」でアイコン設置。App Store不要 |
| 起動 | ブラウザを開かずアイコンから直接起動 |
| 更新通知 | 起動時に「新しいバージョンがあります → 更新」バナーを表示 |
| 更新フロー | NetlifyにデプロイするだけでService Worker更新が自動配信 |
| PC対応 | ChromeまたはEdgeで同様にインストール可能 |
| 対象アプリ | ナレッジQ&Aアプリ＋既存SATOIプラットフォームの両方 |

> ※ 既存プラットフォーム（community.html）へのPWA追加はmanifest.jsonとService Workerの追加のみ。既存機能への影響なし。

## 5. インフラ構成

pre-launch 6〜7ヶ月の段階ではAWSは不要。低コスト・低設定負荷のサービス構成で進める。

| 役割 | サービス | 月額目安 | 備考 |
|---|---|---|---|
| フロントhosting | Netlify（現状維持） | 無料〜$19 | PWA対応追加 |
| 認証・DB・ログ | Supabase | 無料〜$25 | Firebase代替。設定が素直 |
| エスカレーション通知 | LINE Messaging API | 無料枠で十分 | KatsuさんへDM通知 |
| **合計** | — | **$0〜$44 / 月** | pre-launch想定 |

> ※ ユーザー規模・データ量が増えた段階でAWS移行を検討する。現時点では過剰投資になるため推奨しない。

## 8. 追加アイデア：インタラクティブ・ピッチビューワー

### 8-1. 概要

ピッチ資料（PDF/PPTX）をHTML形式でブラウザ表示し、各スライドに「根拠・ロジックを見る」ボタンと「音声で質問する」ボタンを配置する。スライド自体はシンプルに保ちつつ、タップ・クリックで詳細情報が展開される仕組み。

VC・投資家が「このスライドの数字の根拠は？」「このロジックの詳細は？」と思った瞬間に、その場で答えが出てくる。

> ※ ナレッジQ&Aアプリとは**別アプリ**として独立設計する。VC向けに単体で渡せる形にすることが目的。

---

### 8-2. 機能設計

| 機能 | 内容 |
|---|---|
| スライド表示 | PDFをHTML上でページごとに表示（pdf.js使用） |
| 「？」ボタン | スライドごとに右下に配置。タップで根拠・ロジックが右ドロワーから展開 |
| 音声質問 | マイクボタンで話しかけるとAIが音声で回答（Web Speech API使用） |
| 詳細ドロワー | 右側からスライドインして根拠・補足データ・関連数字がバーッと表示 |
| チャット入力 | 音声が難しい環境向けにテキスト入力も併設 |
| ページ連動 | 現在表示中のスライド番号をAIに自動渡し。「このページについて」が自動で文脈になる |

---

### 8-3. UXフロー

1. URLを受け取ったVCがブラウザで開く（アカウント不要・URLのみでアクセス）
2. ピッチ資料がスライド形式でブラウザに表示される
3. 気になるスライドで「？」をタップ → 右側に根拠・ロジックの詳細が展開
4. さらに詳しく知りたい場合はテキストまたは音声で質問
5. AIがそのスライドの文脈を踏まえて即座に回答
6. 「それでも分からない・直接聞きたい」→ エスカレーションボタンでKatsuさんに通知

---

### 8-4. コンテンツ管理（ナレッジの持たせ方）

スライドごとに対応する補足コンテンツをJSONで管理する。

```
pitch-viewer/
├── index.html          # ビューワー本体
├── pitch.pdf           # ピッチ資料PDF
└── knowledge.json      # スライドごとの補足情報
    {
      "slide_1": { "title": "事業概要", "detail": "...", "source": "..." },
      "slide_2": { "title": "市場規模", "detail": "...", "source": "..." },
      ...
    }
```

PDFを差し替えてknowledge.jsonを更新するだけで最新版に切り替え可能。

| 項目 | ナレッジQ&Aアプリ | ピッチビューワー |
|---|---|---|
